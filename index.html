My Coding Story

I first came to coding in March 2019, when a friend told me it was 'literally the best job in the world'. What a sell – 'This is something I've got to try!' I thought.

So I started by signing up to Codecademy.com (on recommendation) and spent the next few months waking up first thing in the morning to do their Computer Science course before my day job. I learnt the basics of Python 3, core software tools such as git and the command line and basic algorithms and programming principles.

I found I had very little motivational issues (coding is really fun!) and even spent my lunch breaks working on problems I'd struggled with in the morning. Next I came up with my own challenges outside of the codecademy's suggestions. I also joined a coding club in the office (which was criminally under-attended).

Over the autumn I decided my job exit strategy would be to go on an adventure with my partner – we'd go travel South America and make our way to New Zealand where she had family and citizenship. I set aside learning code in favour of learning Spanish and things went a little quiet on the coding front. I decided that I could enrol in a development course when I got to New Zealand.

Jump forward to April 2020, where I found myself locked down in New Zealand (having only just made it into the country) with very little to do. It seemed a perfect opportunity to pick up coding again.

At the time Colt Steele had started a Daily Code Challenge about web development, so I started following each day. However, the pace of the course wasn't speedy enough for my needs. I ended up buying a few of his Udemy courses for the sum total of around US$30 and powered through them over the course of the next few weeks. This introduced me to a much wider range of content, from HTML, CSS and Javascript, to Node.js, npm and Express. Much of what I did you'll find below.

However, soon lockdown came to an end and finances became an issue, so we packed our bags and headed over to Te Puke in the Bay of Plenty where we could work in Level 3 picking kiwifruit. Since then, I've been trying to maintain my coding skills as I've travelled through the North Island, doing a little learning here and there, all while working to fund my travels.

Now, having arrived back in Auckland, I'm looking for some more permanent work that will help me develop some professional coding experience!

Python Dice

  • To the code

This was the first programme I wrote by myself. I decided I wanted to use my newfound python skills to produce a terminal app that would give you the result of a dice roll (I actually wrote the initial code on iOS notes using my phone). You can select any number of dice (more than zero that is) and you will receive a line of text telling you your results.

Having got the core input and output functionality working, I spent a fair amount of time thinking of ways that someone could break the programme, and added in catches to prevent user input error. This included someone trying to roll less than one die, or inputting letters rather than numbers. Negative numbers would be converted to their positive equivalent (-1 => 1) and words would throw an input error.

Colt's Colour Game

  • To the code

The aim of this project was to make a simple game, in which players would have to choose the colour that matched the RGB code given in the header. The original game could be made harder or easier by increasing or reducing the number of tiles to choose from.

To improve the game I adjusted the difficulty system. Realising that the diffculty came in how similar the colours were, rather than the number of them, I rewrote the colour generation function so that squares were more or less similar depending on whether the game was in 'hard' or 'easy' mode. The original script simply picked colours by randomly generating an RGB code.

I achieved this by regulating how the rgb code for each square was generated. The different modes are based around the principle of dividing 255 (the maximum value for an rgb colour) into segments and then generating each rgb code from within one of these segments. For easy mode, the programme generates an array containing numbers 1 to 18 in a random order. Each new rgb value is created from the first number popped off the random array. i.e. If the first number in the array is 1, the first rgb value will be between 0 and 14, if the second number popped off is then 18, the next rgb value will be between 241 and 255. This means each value is from a different segment of 255 and the colours produced have varying hue, saturation and brightness.

For the hard mode, the programme divides 255 into four segments and creates an array containing three of the numbers zero to three in a random order. Each colour value is then generated from the order of those segments. For example, a random array of [2, 1, 4] would generate colours in the range rgb(64–127, 0–63, 193–255). This will produce colours that are more similar.

I wanted to keep the option of playing with either three or six squares, so I added buttons to control the number of squares that would appear.

Colt's YelpCamp

  • To the code

YelpCamp is a fully-functional review site, enabling users to sign up and post campsites and comments on other people's uploads. This process taught me the basics of full stack development, looking at both the backend server code, using frameworks (Node.js), libraries (Bootstrap) and databases (MongoDB).

The initial website only supported image urls, so I implemented an image upload system using cloudinary, allowing users to upload their own imagery. I also added animations, including a loading graphic that runs during the upload process. An admin user is always useful to moderate uploads, so I created a user that could edit and delete posts regardless of who created them. Finally, I adjusted the flash messages so that they showed up on login, as well as just sign-up and logging out.

My additions to the app gave me the opportunity to learn about asynchronous javascript functions. I got a large amounts of unhandled promise rejection errors, before realising that while the asynchrousnous functions I'd added all had catches – the code initially used as part of the code-along didn't!
